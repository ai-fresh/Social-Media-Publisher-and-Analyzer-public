Planer publikacji

Monthly calendar view for scheduling posts. APScheduler runs in the background with automatic 3x retry — jobs survive backend restarts.

Calendar view

The Planer view shows a monthly calendar. Key UI elements:

  • Left/right arrows — navigate between months
  • Dziś button — jump to the current month
  • Click any date to see the posts scheduled for that day in the right panel
  • The right panel shows the selected date, the number of posts, and lists each scheduled post (or "Brak zaplanowanych publikacji" if none)

Scheduling a post

From the video detail sheet, click Zaplanuj publikację to schedule the video for a future date. This creates a scheduled post record with pending status.

Prerequisite: The video must have a title and description before it can be scheduled. Run the AI pipeline first (Transkrypcja → Metadane → Okładka).

How the backend scheduler works

APScheduler polls every 60 seconds for pending posts whose scheduled_at time has passed. When a post is due, the scheduler publishes it to the configured platforms and updates the status.

Restart-safe: Jobs are stored in PostgreSQL via SQLAlchemyJobStore. Backend restarts don't lose scheduled posts.

Retry behavior

If publishing fails, the scheduler retries automatically up to 3 times. After 3 failures, the post status changes to failed and no further retries occur.

Backend endpoints

GET    /api/schedule          # list all scheduled posts
POST   /api/schedule          # create a scheduled post
DELETE /api/schedule/{id}     # cancel a scheduled post
