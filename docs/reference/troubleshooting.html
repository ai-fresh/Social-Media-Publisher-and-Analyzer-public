Troubleshooting

Common issues and how to fix them — organized by category.

AI & Transcription

🔑
Transcription fails — "No API key found"

Cause: OPENAI_API_KEY is not set or the key is invalid.

Fix: Add your key to backend/.env:

OPENAI_API_KEY=sk-proj-...

Or set it in the app: Settings → API Keys → OpenAI API Key. Keys set in the app take priority over the environment variable.

⏳
Transcription is stuck on "processing"

Cause: Background task may have crashed silently. Check backend logs.

Fix:

docker compose logs api --tail 50

If the task failed, the video status reverts to uploaded — just retry transcription.

🖼️
Cover generation says "Imagen 4 requires paid plan"

Cause: Google Imagen 4 is not available on the free AI Studio tier.

Fix option 1: Upgrade your Google AI Studio account at aistudio.google.com.

Fix option 2: Switch to DALL-E 3 as the thumbnail provider. In the app: Settings → API Keys → Thumbnail Provider → DALL-E 3. DALL-E 3 will be used automatically as a fallback.

❌
Metadata generation fails — "Transcription required"

Cause: You must run transcription before generating metadata.

Fix: Click Transcribe first, wait for the video status to change to ready, then generate metadata. Or use Generate All which handles the order automatically.

OAuth & Social Accounts

🔗
OAuth window opens but redirects to error page

Cause: The redirect URI in your developer console doesn't match BASE_URL in .env.

Fix: Make sure the redirect URI in your Google/Facebook/TikTok console exactly matches:

{BASE_URL}/api/accounts/oauth/{platform}/callback

For local development: http://localhost:8000/api/accounts/oauth/youtube/callback

🔄
Instagram account disconnects after a few hours

Cause: Instagram short-lived tokens expire. The backend should automatically refresh using the long-lived token, but the Facebook App must have the instagram_basic and pages_read_engagement permissions approved.

Fix: Check your Facebook App's permissions in the developer console. Reconnect the account after granting missing permissions.

🎵
TikTok publishing fails — "sandbox only"

Cause: TikTok Content Posting API requires explicit approval for production use.

Fix: Apply for production access in your TikTok developer console. Until approved, all posts go to sandbox (visible only in TikTok sandbox environment).

Docker & Backend

🐳
Backend won't start — "could not connect to server"

Cause: PostgreSQL container hasn't finished starting yet, or DATABASE_URL is incorrect.

Fix:

docker compose ps               # check container statuses
docker compose logs db          # check postgres logs
docker compose restart api      # restart the API after DB is ready
📁
Uploads fail — "Permission denied" on the uploads directory

Cause: The UPLOAD_DIR or THUMBNAIL_DIR doesn't exist or the API container doesn't have write permissions.

Fix:

docker compose exec api mkdir -p /app/uploads /app/thumbnails
docker compose exec api chmod 755 /app/uploads /app/thumbnails
⚡
ffprobe not found — video duration not detected

Cause: ffmpeg is not installed in the container. The Dockerfile includes it, but custom builds may omit it.

Fix: The Dockerfile uses python:3.11-slim with apt-get install ffmpeg. Rebuild the container:

docker compose build --no-cache api
docker compose up -d

macOS App

🍎
App won't connect to backend — "Connection refused"

Cause: The backend is not running, or the app is pointing to the wrong URL.

Fix: Confirm the backend is running with docker compose ps. The macOS app connects to http://localhost:8000 by default. For remote servers, update APIClient.swift:

private let baseURL = "https://your-server.com"
🎤
Microphone not working for voice input

Cause: macOS microphone permission not granted to the app.

Fix: Go to System Settings → Privacy & Security → Microphone and enable access for Social Media Publisher.

Check backend logs first: Most issues are logged in the FastAPI backend. Run docker compose logs api -f while reproducing the issue to see detailed error messages.
