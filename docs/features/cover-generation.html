Cover Generation

Step 3 of the AI pipeline — generate a video thumbnail using Imagen 4 (Google) or DALL-E 3 (OpenAI), with selectable aspect ratio.

In the app

Cover generation is Step 3 in the video detail sheet. Before clicking the generate button, select the Proporcje (aspect ratio) using the segmented control:

  • 9:16 — vertical, for TikTok and Instagram Reels
  • 16:9 — landscape, for YouTube
  • 1:1 — square, for Instagram feed posts

When complete, the step shows a green checkmark. The generated thumbnail is shown in the video preview area at the top of the sheet.

Prerequisite: Transcription (Step 1) and Metadata generation (Step 2) must complete before cover generation. Use Generate All to run all three steps at once.

Providers

🎨 Imagen 4 (Google)

Google's Imagen 4 model via Gemini API. Default provider when configured.

  • Requires GEMINI_API_KEY
  • Requires paid Google AI Studio plan

🖌️ DALL-E 3 (OpenAI)

OpenAI's DALL-E 3 image model. Used when Imagen 4 is not configured.

  • Requires OPENAI_API_KEY
  • Output resized to 1280×720 via Pillow

Switching providers

The active thumbnail provider is set in Konta → Ustawienia marki in the app. The toggle shows Imagen 4 (Google) and DALL-E 3 (OpenAI). See Brand Settings for details.

Style prompt

The image prompt sent to the model is automatically constructed from the video transcript, title, description, and the Prompt stylu okładki field configured in Konta → Ustawienia marki. A good style prompt produces visually consistent thumbnails across all your videos.

Example style prompt: "Cinematic dark background, bold white sans-serif text, high contrast, professional YouTube thumbnail aesthetic."

Backend endpoint

POST /api/videos/{video_id}/generate-cover

The request body includes the selected aspect ratio. Cover generation runs as a background task — the video status changes to processing while the image is being generated.
